# Справочник по конфигурационным файлам

Avtomatika использует файлы формата [TOML](https://toml.io/en/) для управления доступом и настройками клиентов и воркеров.

Система реализует принцип **Fail Fast**: если при запуске обнаружена синтаксическая ошибка TOML или отсутствуют обязательные поля, Оркестратор **не запустится** и выдаст исключение. Это гарантирует, что система не будет работать в некорректном или небезопасном состоянии.

---

## 1. clients.toml

Этот файл определяет список API-клиентов, которым разрешено создавать задания.

**Путь по умолчанию:** Можно задать через переменную окружения `CLIENTS_CONFIG_PATH`. Обычно находится в корне проекта.

### Структура

Файл состоит из секций (таблиц), где имя секции — это уникальный идентификатор клиента (для логов и удобства).

| Поле | Тип | Обязательно | Описание |
| :--- | :--- | :--- | :--- |
| `token` | String | **Да** | Секретный токен, который клиент должен передавать в заголовке `X-Client-Token`. |
| `plan` | String | Нет | Название тарифного плана (например, "free", "premium"). Используется в блупринтах для логики. |
| `monthly_attempts` | Integer | Нет | Месячная квота на количество запросов. Если задана, Оркестратор будет отслеживать и блокировать запросы сверх лимита. |
| `*` | Любой | Нет | Любые другие поля (например, `languages`, `callback_url`) будут доступны в `context.client.params`. |

### Пример

```toml
# Премиум клиент
[client_premium_user]
token = "sec_vip_token_123"
plan = "premium"
monthly_attempts = 100000
# Кастомные параметры
languages = ["en", "de", "fr"]
priority_support = true

# Бесплатный клиент
[client_free_user]
token = "sec_free_token_456"
plan = "free"
monthly_attempts = 100
languages = ["en"]
```

## 2. workers.toml

Этот файл используется для настройки индивидуальной аутентификации воркеров. Это более безопасная альтернатива использованию одного глобального токена.

**Путь по умолчанию:** Можно задать через переменную окружения `WORKERS_CONFIG_PATH`.

### Структура

Файл состоит из секций, где имя секции должно **точно совпадать** с `worker_id`, который воркер использует при регистрации.

| Поле | Тип | Обязательно | Описание |
| :--- | :--- | :--- | :--- |
| `token` | String | **Да** | Индивидуальный секретный токен для этого воркера. |
| `description` | String | Нет | Описание воркера для администраторов. |

### Особенности безопасности
*   При запуске Оркестратор вычисляет SHA-256 хэш токена и хранит в памяти (Redis) только хэш. Оригинальный токен нигде не сохраняется.
*   При входящем запросе от воркера хэш переданного токена сравнивается с сохраненным.
*   **Оптимизация:** Хэши токенов кэшируются в памяти процесса для минимизации нагрузки на CPU при частых Heartbeat.

### Пример

```toml
# Имя секции должно совпадать с WORKER_ID на стороне воркера
[gpu-worker-01]
token = "super-secret-token-for-gpu-01"
description = "Основной GPU узел для рендеринга видео"

[cpu-worker-01]
token = "another-secret-token-for-cpu-01"
description = "Воркер общего назначения на CPU"
```

## 3. schedules.toml

Этот файл настраивает нативный планировщик (Native Scheduler), позволяя запускать блупринты периодически без внешних заданий cron.

**Путь по умолчанию:** Можно задать через переменную окружения `SCHEDULES_CONFIG_PATH`.

### Структура

Каждая секция представляет собой запланированную задачу. Имя секции служит уникальным идентификатором задачи.

| Поле | Тип | Обязательно | Описание |
| :--- | :--- | :--- | :--- |
| `blueprint` | String | **Да** | Имя блупринта для запуска. |
| `input_data` | Dictionary | Нет | Исходные данные для задания. По умолчанию пустой словарь. |
| `dispatch_timeout` | Integer | Нет | Таймаут очереди (секунды). Задача отменяется, если не взята воркером за это время. |
| `result_timeout` | Integer | Нет | Дедлайн выполнения (секунды). Абсолютное время с момента создания для получения результата. |
| `interval_seconds` | Integer | *Одно из* | Запускать задачу каждые N секунд. |
| `daily_at` | String | *Одно из* | Запускать ежедневно в определенное время ("HH:MM"). |
| `weekly_days` | List[String] | *Одно из* | Запускать в определенные дни недели ("mon", "tue", ...) в указанное время `time`. |
| `monthly_dates` | List[Integer] | *Одно из* | Запускать в определенные числа месяца (1-31) в указанное время `time`. |
| `time` | String | *Одно из* | Обязательно для `weekly_days` и `monthly_dates`. Формат "HH:MM". |

**Примечание по часовым поясам:** Все поля времени интерпретируются согласно глобальной переменной окружения `TZ` (по умолчанию "UTC").

### Пример

```toml
[cleanup_job]
blueprint = "system_cleanup"
interval_seconds = 3600
input_data = { target = "temp_files" }

[daily_report]
blueprint = "generate_report"
daily_at = "09:00" # Запуск в 09:00 по времени TZ
input_data = { type = "full_daily" }

[weekly_backup]
blueprint = "full_backup"
weekly_days = ["fri"]
time = "23:00"
input_data = { compression = "max" }
```

## Динамическая перезагрузка

Вы можете обновить `workers.toml` без перезапуска Оркестратора.
Для этого отправьте POST-запрос (с валидным токеном клиента) на эндпоинт:

`POST /api/v1/admin/reload-workers`

Это заставит Оркестратор перечитать файл и обновить хэши токенов в Redis.

## Переменные окружения

Помимо конфигурационных файлов, Оркестратор настраивается через переменные окружения.

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `API_HOST` | Хост для запуска API-сервера. | `0.0.0.0` |
| `API_PORT` | Порт для запуска API-сервера. | `8080` |
| `ENABLE_CLIENT_API` | **Pure Holon Mode:** Если `false`, отключает Client API (`/api/v1/...`). Принимает задачи только от родителя через RXON. | `true` |
| `REDIS_HOST` | Адрес сервера Redis. Обязательно для production. | `""` (MemoryStorage) |
| `REDIS_PORT` | Порт Redis. | `6379` |
| `REDIS_DB` | Индекс базы данных Redis. | `0` |
| `INSTANCE_ID` | Уникальный ID экземпляра. Используется как имя потребителя в Redis Streams. | `hostname` |
| `CLIENT_TOKEN` | Глобальный токен для API-клиентов (резервный). | `secure-orchestrator-token` |
| `GLOBAL_WORKER_TOKEN` | Глобальный токен для воркеров (резервный). | `secure-worker-token` |
| `CLIENT_API_PREFIX` | **Настраиваемый путь API:** Сегмент URL для клиентского API. Если пуст, API будет в корне. | `api` |
| `WORKERS_CONFIG_PATH` | Путь к `workers.toml`. | `""` |
| `CLIENTS_CONFIG_PATH` | Путь к `clients.toml`. | `""` |
| `SCHEDULES_CONFIG_PATH` | Путь к `schedules.toml`. | `""` |
| `BLUEPRINTS_DIR` | Путь к директории с Python-файлами блупринтов. | `""` |
| `TZ` | **Глобальный часовой пояс:** Влияет на планировщик и логи (например, "Europe/Moscow"). | `UTC` |
| `LOG_LEVEL` | Уровень логирования (`DEBUG`, `INFO`, `WARNING`, `ERROR`). | `INFO` |
| `LOG_FORMAT` | Формат логов (`text` или `json`). | `json` |
| `WORKER_TIMEOUT_SECONDS` | Дедлайн выполнения задачи по умолчанию. | `300` |
| `WORKER_POLL_TIMEOUT_SECONDS` | Таймаут Long-polling запросов от воркеров. | `30` |
| `WORKER_HEALTH_CHECK_INTERVAL_SECONDS` | Интервал обновления TTL воркера (health check). | `60` |
| `JOB_MAX_RETRIES` | Максимальное количество попыток при временных сбоях. | `3` |
| `WATCHER_INTERVAL_SECONDS` | Интервал проверки таймаутов фоновым процессом Watcher. | `20` |
| `WATCHER_LIMIT` | Максимальное количество проверяемых таймаутов за один цикл. | `500` |
| `STRICT_EVENT_VALIDATION` | Если `true`, отклоняет события воркеров, не описанные в схемах. | `true` |
| `REPUTATION_MIN_THRESHOLD` | Минимальная репутация воркера для назначения задач. | `0.3` |
| `REPUTATION_PENALTY_CONTRACT_VIOLATION` | Штраф за нарушение контракта данных. | `0.2` |
| `REPUTATION_PENALTY_TASK_FAILURE` | Штраф за критический или постоянный сбой задачи. | `0.05` |
| `REPUTATION_REWARD_SUCCESS` | Бонус за каждую успешно выполненную задачу. | `0.001` |
| `EXECUTOR_MAX_CONCURRENT_JOBS` | Лимит одновременных задач (обработчиков) в Оркестраторе. | `1000` |
| `DISPATCHER_SOFT_LIMIT` | Порог очереди воркера перед переливом на дорогих кандидатов. | `3` |
| `DISPATCHER_MAX_CANDIDATES` | Лимит проверки подходящих воркеров при фильтрации (защита от O(N)). | `50` |
| `WORK_STEALING_ENABLED` | Разрешить свободным воркерам атомарно красть задачи у других. | `true` |
| `HISTORY_DATABASE_URI` | URI хранилища истории (`sqlite:///...` или `postgresql://...`). | `""` (Отключено) |
| `RATE_LIMITING_ENABLED` | Включить ограничение частоты запросов. | `true` |
| `MAX_TRANSITIONS_PER_JOB` | Защита от бесконечных циклов в блупринтах. | `100` |
| `WORKER_AUTH_MODE` | **Режим аутентификации:** `mixed` (токены+mTLS), `mtls-only` (строгий mTLS + STS), `token-only`. | `mixed` |

### Безопасность и TLS (mTLS)

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `TLS_ENABLED` | Включить HTTPS сервер. | `false` |
| `TLS_CERT_PATH` | Путь к SSL сертификату сервера (`.crt`). | `""` |
| `TLS_KEY_PATH` | Путь к приватному ключу сервера (`.key`). | `""` |
| `TLS_CA_PATH` | Путь к CA бандлу для проверки сертификатов клиентов. | `""` |
| `TLS_REQUIRE_CLIENT_CERT` | Если `true`, сервер будет требовать валидный сертификат (mTLS). | `false` |
| `REDIS_ENCRYPTION_KEY` | **Шифрование в Redis:** Если задан, токены воркеров шифруются (AES-GCM). | `""` |

### S3 Хранилище (Опционально)

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `S3_ENDPOINT_URL` | URL S3-совместимого хранилища. Обязательно для включения S3. | `""` |
| `S3_ACCESS_KEY` | Access Key ID. | `""` |
| `S3_SECRET_KEY` | Secret Access Key. | `""` |
| `S3_MAX_CONCURRENCY` | Лимит одновременных соединений с S3 для всего кластера. | `100` |
| `S3_AUTO_CLEANUP` | Если `true`, автоматически удаляет файлы S3 при завершении задания. | `true` |
